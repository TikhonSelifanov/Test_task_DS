## Финальная модель

**Алгоритм:** CatBoostClassifier с подбором гиперпараметров через Optuna.

**Признаки:**
- 8 исходных категориальных признаков
- 25 числовых признаков после feature engineering (платежи, отношения к доходу, группы риска, взаимодействия)

**Конвейер:**
1. Загрузка данных и feature engineering (`load_and_engineer`)
2. Стратифицированное разделение train/test (80/20)
3. 5-fold CV с Optuna по метрике `average_precision`
4. Обучение лучшей модели на полном train
5. Подбор порога классификации по F1

## Бизнес-правила

Модель дополнена бизнес-правилами для улучшения качества:

### Автоотказ
- **Платёж > 50% дохода** — автоматический отказ
- **Кредит > 10 * годового дохода** — автоматический отказ

### Корректировки риска (на основе анализа данных)
- **Молодой клиент (<25) + низкий доход (<20k)** - +10% к риску
- **Среднее образование** - +8% к риску
- **Короткий срок кредита (<6 мес)** - -5% от риска
- **Высокий доход (>60k)** - -5% от риска
- **Санкт-Петербург/ЛО регион** - -4% от риска

### Защита от ложных срабатываний
- **Микрокредиты (< 5000 руб., платёж < 5% дохода)** — риск ограничен на 5%

## Стабильность

- PSI скор: 0.028
- PSI ключевых признаков < 0.025
- Качество модели не деградирует по месяцам

## Примечание по качеству

После тестирования логистической регрессии, случайного леса, XGBoost, CatBoost, SMOTE, WOE-кодирования лучший результат — PR-AUC примерно 0.30–0.31. Что говорит о том, что текущих данных недостаточно. Для улучшения результата необходимы дополнительные данные. Бизнес-правила, созданные исходя из логики и анализа данных, были полезны только для Recall, в совокупности они смогли повысить Recall примерно на 20%
